2. Typescript

The typescript file( * .ts) will be compiled into js file with same name( * .js), as soon as you save the typescript file.

2.1 With the help of declaration code, any legal js code is legal typescript code.

That means, if you copy a legal js code to typescript file, the code is legal too. Therefore, typescript is considered to be the superset of javascript. And the generated js file is: Most common js libraries' declaration for typescript can be installed by [nuget](http://www.nuget.org/packages?q=DefinitelyTyped) or found on [DefinitelyTyped](https://github.com/borisyankov/DefinitelyTyped).

2.2 It's an Object-Oriented language, a class can include constructor and private members, and a class can be included in a module.

2.3 It supports "Rename", so it is much more maintainable than common js.

2.4 It has a better intellisense.

2.5 It's a strong-typed language, so it has compile-time type checking.

2.6 It's easy to see the js code that generated by typescript.

3. LESS

The LESS file( * .less) will be compiled into CSS file with same name( * .css), as soon as you save the LESS file.

3.1 Any legal CSS code is legal LESS code.

That means, if you copy a legal CSS code to LESS file, the code is legal too.

3.2 It supports variables.

The left is LESS code, and the right one is the generated CSS file.

3.3 It supports nested rules, so it make CSS code easier to read.

3.4 It supports function, that will reduce some work of cross-browser compatibility.

The left side is LESS file, and the right one is generated CSS file.
